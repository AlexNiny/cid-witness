# Design Context

## Register

Product UI.

## Scene

A builder is preparing a hackathon submission on a laptop in a bright coworking room, switching between docs, repo, and demo. The interface should feel clear, sturdy, and easy to screenshot.

## Color Strategy

Restrained interface with warm paper neutrals, deep green-black text, Filecoin blue for primary actions, and small lime plus rust accents for state and evidence.

## Typography

Use the system UI stack. Keep labels compact, data legible, and prose at readable line lengths.

## Components

- Top status rail with live network stats.
- Search console for the main mechanic.
- Evidence panel with concise rows and generated share text.
- Inline error and empty states, no modal-first flows.
